### MR-012 · 2026-07-22 · data
- **決策**：Phase E 資料模型定案，四點——
  ①**展覽與分類正交**：分類＝媒材（無序篩選、帶主題），展覽＝策展（有序動線＋前言），作品兩者都有；
  ②**持久層收成單一 document** `artwall.library.v2`（works／categories／exhibitions／overrides／hidden），
  帶 `schemaVersion`，匯出 JSON 即該 document 本身，需寫 v1→v2 遷移；
  ③**分類個性綁在紋理上**：使用者只選名稱／顏色／紋理，`easing`／`radius`／`border` 隨紋理成組帶入，
  `accentDark` 由 accent 以 HSL 提亮自動推導並驗對比度；
  ④**展覽入口為站頭第二排**「依媒材／依展覽」模式切換，不新增頁面層級
- **理由**：分類是無序篩選、展覽是有序動線，資料結構本就不同，合併會逼出「展覽也有紋理嗎」這種答不出的問題；
  MR-010 後匯出檔＝正式發佈管道，儲存與匯出格式不一致就是多一層轉換與 bug 溫床，且無 `schemaVersion`
  會讓舊匯出檔在改版後**靜默讀壞**；動效曲線不能暴露 `cubic-bezier` 給非工程師填
- **棄選**：展覽為主軸（推翻 MR-008 已上線的分類主題，且無展覽者首頁全空）；統一成 collection（概念混淆）；
  新分類給固定中性預設（會明顯比內建六類沒個性）；維持多 key 分散（`useLibrary` 已 254 行，只會更腫）
- **影響**：`CategoryId` union → `string`，**編譯期保證消失**——`CATEGORY_MAP[work.category]` 現有 5 處
  直接索引（`WorkCard:17`／`WorkDetail:19`／`works.ts:27,38`／`useAppearance:17`）必須補 fallback。
  刪分類已決定「有作品就擋下」，但**匯入 JSON 仍會帶進孤兒分類**，故防禦不可省。
  `EditorPanel`（610 行）拆為分頁式（作品／分類／展覽／站台），屬 E 的成本非額外加碼
- **錨點**：待 Phase E PR

### MR-010 · 2026-07-22 · deploy

### MR-009 · 2026-07-21 · data
- **決策**：瀏覽器內編輯只做**本機層**——圖片存 IndexedDB、文字與站台設定存 localStorage，
  透過「匯出 JSON」把資料帶進開發流程才會成為訪客看得到的版本。上傳一律壓成縮圖 800px
  與詳情用圖 1800px 兩份。另加整體背景切換（白盒／暖紙／清水模／暗展廳），與分類主題正交
- **理由**：沒有後端就沒有「對外生效」的路，硬做會給出「改完就上線」的錯覺（面板上明講此限制）。
  圖片不能放 localStorage：5MB 上限且只能存字串，一張手機照轉 base64 就吃掉大半
- **棄選**：全部塞 localStorage（配額炸）；直接接後端／GitHub API 寫回（此刻無真資料，等於先付架構成本）；
  上傳存原始檔（4000px 照片直接進牆面）
- **影響**：解決原待討論事項 #1 的「深」；新待討論事項改為「對外生效」的路徑選擇
- **錨點**：PR #4

### MR-008 · 2026-07-21 · wall
- **決策**：作品牆設計定案。①開場＝3.4 秒幾何動畫，Skip 常駐／只播一次／減少動態時不播，
  真互動探索留彩蛋；②分類切換取代顏色切換，換 accent＋動效曲線＋卡片邊緣＋背景紋理，
  且 accent 貫穿站頭底線、卡片外框、進度條、頁尾；③版位以 `aspect-ratio` 鎖死三種比例，
  寬螢幕水平長廊（軌道高度反推卡片寬）、窄螢幕網格
- **理由**：探索感要在開頭但不能擋路（30 秒法則）；紋理灰階、不透明度 ≤3%，且每個分類補
  `accentDark`——墨色與石膏灰在暗底會整個消失；長廊以軌道高度為唯一尺寸來源，
  交叉懸掛才不可能把版面推爆
- **棄選**：整站隨分類換皮（等於維護 4 套設計系統，且吃作品顏色）；JS masonry（跑版元凶）；
  平板兩欄（單張圖大到要捲動才看得完）
- **影響**：覆蓋 MR-005 琥珀單色；MR-001／MR-002／MR-004 的迷宮實作全數移除；移除 nes.css 依賴
- **錨點**：PR #4

### MR-006 · 2026-07-17 · maze
- **決策**：視覺重整為磚塊地牢 —— 走廊與房間改以磁磚地板 + 有厚度的牆渲染，取代目前的 SVG 粗線條；迷霧起手改為「全圖輪廓可見、內容未探明」
- **理由**：Phase 1 實測畫面被評為醜且無遊戲感。根因是把地牢畫成「線」而非「空間」：走廊 stroke-width 11 像水管、無地板無牆無厚度、網格背景在重寫時遺失；迷霧起手只露一格，訪客看到近乎空白，像壞掉而非待探索
- **棄選**：只做採光與質感微調（偷不走地牢感）；換掉俯瞰視角改側面剖面或第一人稱（等於 Phase 1 重做）；改回六彩房間色（單色不是問題，缺明暗層次才是 —— 層次改由地板／牆／陰影／發光強度提供，見 MR-005）
- **影響**：MazeMap 重畫；`useMaze` / `useExploration` 邏輯層不動 —— 方向鍵移動、進度計數、繞徑皆經實測驗證正確
- **錨點**：PR #2
