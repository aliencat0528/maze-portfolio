## 決策日誌（最近 10 筆，新的在上）

### MR-005 · 2026-07-17 · styles
- **決策**：色調改琥珀單色終端機 —— 基準色 `#ffb000`；六房間識別色改為同色系微幅色溫差，取代原本的高彩度異色
- **理由**：統一的復古 CRT 質感；純粹單色會讓六房間完全無法辨識，故以琥珀色溫差保留最低限度識別
- **棄選**：維持 CRT 綠 `#14fdce` + 六高彩度房間色；純單色零房間差異
- **錨點**：待 Phase 1 PR

### MR-004 · 2026-07-16 · maze
- **決策**：成就改為網站內行為解鎖（走遍全圖 / 尋獲密室 / 看完作品 / 速通 / Konami），進度存 localStorage
- **理由**：現況 `unlocked` 由資料寫死，「隱藏成就」無任何解鎖行為；localStorage 提供回訪動機（目前 0 個 localStorage，重整歸零）
- **棄選**：維持靜態成就清單 —— 有成就的外觀但無互動價值
- **錨點**：待 Phase 2 PR

### MR-003 · 2026-07-16 · deps
- **決策**：移除 `gsap`、`@vueuse/core`；`nes.css` **實際採用**，但 import 來源改 `nes-core.min.css`（288K → 52K）。同時補 ESLint + `npm run lint`
- **理由**：gsap/@vueuse 引用數為 0；nes.css 的 288K 中約 236K 是 Mario/Kirby/寶可夢等像素精靈圖，對履歷無用，`nes-core` 保留全部 UI 元件；根 CLAUDE.md 要求 commit 前 lint，但該 script 不存在
- **棄選**：整包移除 nes.css（改為實際採用其像素元件結構）；沿用 `nes.min.css`（236K 精靈圖純浪費）
- **錨點**：待 Phase 1 PR

### MR-002 · 2026-07-16 · maze
- **決策**：迷宮層重寫 —— `connections` 為唯一真相、刪除 MazeMap 硬編座標；採地牢格子圖（房間為方塊、走廊為可走通道，房間座標 ×2 展開以留出走廊空間，走廊由 connections 以 L 形繞徑生成）；迷霧規則為「已訪問全亮／鄰接暗輪廓／其餘全黑」；桌機方向鍵移動，手機點擊後 BFS 自動尋路
- **理由**：現況 entrance 直連全部 6 房（`MazeMap.vue:35-45`），繞過 `connections`；`position` 與 SVG 像素座標兩套真相且 `position` 為死欄位；探索進度因此無意義
- **棄選**：完整 tile 迷宮（有牆有死路會真的迷路，對 HR 是災難）；虛擬搖桿（手機體驗差，而履歷流量大半在手機）；維持星形選單（無遊戲性）
- **錨點**：待 Phase 1 PR

### MR-001 · 2026-07-16 · maze
